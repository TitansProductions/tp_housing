**TP Housing**

# What has been updated?

## Changelog - (Added, Fixed, Removed):

1. Added the possibility when a location is about to set (wardrobe, storage, etc), to also have an extra option to cancel action.
2. Added in the config (2) new options that allow you to open storage or wardrobe through the menu than having locations with prompts.

## Modified Files:

- client/tp-client_menu.lua
- locales.lua
- fxmanifest.lua

## Notes:

1. Add new locales on `locales.lua` file:

```lua
['SET_MENU_CANCEL'] = "Press ~e~[DEL]~q~ To cancel location placement.", -- 1.3.2
['MENU_WARDROBE_ACCESS_LOCATION'] = "Checkout Wardrobe", -- 1.3.2
['MENU_WARDROBE_ACCESS_LOCATION_DESCRIPTION'] = "", -- 1.3.2

['MENU_STORAGE_LOCATION_ACCESS'] = "Access Storage", -- 1.3.2
['MENU_STORAGE_LOCATION_ACCESS_DESCRIPTION'] = "", -- 1.3.2
```

2. Add the (2) new configuration options on `config.lua` file related to storage and wardrobe menu access.

```lua
-- Set to false if you want the wardrobe access to be through a location instead through the menu.
Config.DisableWardobePositionSet = true -- 1.3.2

-- Set to false if you want the storage access to be through a location instead through the menu.
Config.DisableStoragePositionSet = true -- 1.3.2
```
